## Authentication {#authentication}

All API requests require authentication using your API key.

<Callout type="info">
Get your API key from the Developer Settings in your dashboard:
<a href="https://app.salsaasaservice.com/settings/api">Developer Settings</a>
</Callout>

### Authorization Header

~~~http
Authorization: Bearer YOUR_API_KEY
~~~

**cURL**
~~~bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.salsaasaservice.com/v2/ingredients
~~~

**Fetch**
~~~javascript
fetch("https://api.salsaasaservice.com/v2/ingredients", {
  headers: { Authorization: "Bearer YOUR_API_KEY" }
});
~~~

## Rate Limits {#rate-limits}

- **Free Plan**: 1,000 requests per hour  
- **Professional Plan**: 10,000 requests per hour  
- **Enterprise Plan**: 100,000 requests per hour  

Rate limit headers are included in all responses:

~~~http
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
~~~

<Callout type="warning">
If youâ€™re close to the limit, back off and retry after <code>X-RateLimit-Reset</code>.
</Callout>

## Inventory Management {#inventory}

### Get All Ingredients

~~~http
GET /ingredients
~~~

**Response**
~~~json
{
  "data": [
    {
      "id": "ing_123",
      "name": "Roma Tomatoes",
      "category": "vegetables",
      "current_stock": 25.5,
      "unit": "lbs",
      "quality_score": 8.5
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 50,
    "total": 1
  }
}
~~~

### Update Ingredient Stock

~~~http
PUT /ingredients/{ingredient_id}/stock
~~~

**Request**
~~~json
{
  "quantity": 30.0,
  "unit": "lbs",
  "notes": "New shipment received"
}
~~~

## Analytics & Reporting {#analytics}

### Get Production Analytics

~~~http
GET /analytics/production?period=30d
~~~

**Response**
~~~json
{
  "period": "30d",
  "total_batches": 45,
  "average_quality_score": 8.7
}
~~~

## Error Handling {#error-handling}

**Error format**
~~~json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request body is missing required fields"
  }
}
~~~

<Accordion title="Client Errors (4xx)">
- `400` Bad Request  
- `401` Unauthorized  
- `403` Forbidden  
- `404` Not Found  
- `422` Validation error  
- `429` Too Many Requests  
</Accordion>

<Accordion title="Server Errors (5xx)">
- `500` Internal Server Error  
- `502` Bad Gateway  
- `503` Service Unavailable  
</Accordion>
